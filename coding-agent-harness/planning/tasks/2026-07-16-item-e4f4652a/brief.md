# 验证并接入本机视觉文字定位

## Task ID

`2026-07-16-item-e4f4652a`

## 创建日期

2026-07-16

## 一句话结果

用真实 macOS 窗口验证 Apple Vision 的本机文字与区域识别；达到门槛后，把它作为 UI Bridge 的跨平台视觉文字后备能力接入。

## 完成后能得到什么

用户可以让 UI Bridge 在无障碍树不完整时，直接在本机从当前窗口截图中查找文字及其区域，
无需把整张截图交给云端模型。能力先用真实中文界面验证速度、文字命中和区域有效性；通过后
以通用结果格式接入，macOS 使用 Apple Vision，未来 Windows 可替换成系统对应实现。

## 交付物

- 可见产物：本机视觉文字识别自检结果，以及通过 MCP/HTTP 可读取的文字区域候选。
- 修改位置：`UIBridgeProtocol`、`UIBridgeMacCore`、服务与 MCP 接入、Skill 和文档。
- 验证证据：真实窗口截图命中、延迟记录、坐标范围检查、既有回归。

## 第一眼应该看什么

先读 `task_plan.md` 和 `progress.md`，再看本机识别实现与自检输出。

## 边界

- 范围内：真实截图验证、跨平台文字区域模型、macOS Apple Vision 实现、只读查询入口和文档。
- 范围外：本地大视觉模型、图标分类、长期保存截图、自动点击、Windows 实现。
- 停止条件：真实中文界面命中或区域明显不可靠，或者接入必须降低现有安全边界时停止正式接入。

## 完成判断

- 真实窗口中的关键中文文字被识别，返回区域落在窗口内并覆盖可见文字。
- 热运行耗时适合作为按需后备路线，失败时返回空结果而不是猜测。
- 识别只处理当前截图，不保存截图，也不能绕过新快照、计划检查或操作后验证。
- macOS 实现隐藏在通用接口后，未来 Windows 无需改变 Agent 调用方式。
- 构建、协议、核心、安全和 Skill 自检全部通过。
