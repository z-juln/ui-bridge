# UI Bridge：操作经验库计划

状态：产品方向已确认，尚未进入实现。经验库同时服务原生 App、网页和内嵌 WebView，
不是 Web Bridge 的附属功能。

## 1. 目标

让用户或维护者教会 UI Bridge“某个界面中如何稳定找到一个目标、执行一步操作并确认结果”，
以后由 Cursor、WorkBuddy 和其他 Agent 复用，而不是每次从完整界面重新搜索。

经验用于提高发现速度和成功率，不能绕过当前快照、权限、安全确认和操作后验证。

## 2. 经验类型

- 官方经验：随版本发布、经过代表环境验证的应用或网站操作经验。
- 用户经验：用户通过框选、命名和复测保存在本机的经验。
- 建议经验：系统从多次成功操作中生成候选，用户确认后才能成为正式经验。

官方经验、用户经验和建议经验必须标明来源，不能互相静默覆盖。用户可以停用、编辑和删除。

## 3. 一条经验包含什么

- 目标范围：原生应用身份、网站来源、页面类型或 WebView 宿主。
- 操作意图：例如搜索、打开详情、填写收件人或进入设置。
- 使用前条件：窗口或页面应处于什么状态。
- 多组定位依据及优先级。
- 允许执行的动作，不保存任意脚本。
- 操作后的预期变化和验证方法。
- 风险类别，以及是否需要前台或人工确认。
- 来源、版本、可信度、最近成功时间和连续失败次数。

原生 App 的定位依据可以包括控件角色、名称、系统标识、窗口类型、父子关系、附近标签和
菜单路径。网页可以包括角色、标签、稳定属性、链接特征、页面区域、iframe 和 Shadow DOM
关系。所有位置都必须从当前快照重新计算。

## 4. 明确不保存的内容

- 密码、Cookie、令牌、验证码或密码类控件值。
- 固定屏幕坐标、临时控件句柄、DOM 顺序或页面节点编号。
- 完整截图、完整 DOM、聊天正文、表单正文或下载内容。
- 能绕过系统权限、网站权限、验证码或危险操作确认的步骤。

如未来需要视觉模板，必须由用户单独选择区域并确认保存，默认不开启。

## 5. 用户教学流程

1. 用户进入“教 UI Bridge”模式并选择当前 App 窗口或网页。
2. 鼠标经过控件时显示高亮，用户点击或框选目标。
3. UI Bridge 展示识别出的控件含义、作用范围和候选定位依据。
4. 用户命名意图，并选择这是查找经验、普通操作还是危险操作入口。
5. UI Bridge 在不产生业务影响的前提下重新读取、重新定位并高亮复测。
6. 涉及写入时，使用隔离数据或由用户明确触发一次真实复测。
7. 复测成功后保存；无法稳定重现时只保留为草稿。

## 6. 使用和失效机制

1. 根据当前目标和意图查找候选经验。
2. 检查应用、窗口、网站或页面特征是否仍匹配。
3. 用当前快照重新定位，得到可信度和候选控件。
4. 可信度不足时退回通用发现，不盲目执行。
5. 按现有安全规则执行，并重新读取验证结果。
6. 成功后更新最近验证信息；连续失败后自动停用并提示修复。

应用升级、网站改版、窗口变化和页面导航都可能使经验失效。经验只能缩短定位过程，不能让
旧快照、旧坐标或旧控件句柄继续有效。

## 7. 设置与调试界面

新增“经验库”栏目，至少展示：

- 官方、用户和建议经验。
- 适用 App 或网站、意图、可信度、最近成功时间和状态。
- 定位依据、验证条件和风险类别的可读说明。
- 立即复测、编辑、停用、删除、导入和导出入口。
- 当前操控使用了哪条经验、命中了什么控件、为何降级到通用发现。

删除、覆盖、批量导入和启用危险操作经验需要二次确认。

## 8. 客户端边界

- Agent 可以查询和使用已启用的经验。
- Agent 可以提出经验建议，但不能静默保存、覆盖或提高权限。
- 客户端只看到脱敏后的经验说明和使用结果，不获得敏感界面内容。
- 经验不能扩大应用访问范围，也不能取消危险操作确认。
- 本地导入的密码和 Cookie 与经验库完全隔离。

## 9. 实施阶段

### 阶段 A：格式、存储和管理界面

- 建立跨 App/Web 的统一经验格式、版本和本地存储。
- 完成查看、停用、删除、导入导出和隐私检查。

### 阶段 B：原生 App 教学与只读复用

- 支持用户在 App 窗口中框选控件、命名和无副作用复测。
- 优先覆盖查找、读取和导航，不先处理发送或删除。

### 阶段 C：原生 App 安全操作经验

- 保存动作前条件和结果验证。
- 接入前台升级、危险操作确认和连续失败停用。

### 阶段 D：网页教学与跨通道复用

- Web Bridge 使用同一经验格式。
- 同一意图可分别保存内嵌浏览器、现有 Chrome 和 WebView 的定位方式。

### 阶段 E：官方经验与更新

- 建立可签名、可审查的官方经验包。
- 更新前显示差异，允许用户保留本地版本或回退。

## 10. 第一轮完成标准

- 用户能为一个原生 App 控件创建经验，重开窗口后仍能从当前快照正确定位。
- App 更新或布局变化后，错误经验不会盲目执行并能自动降级。
- 每次经验操作仍完成结果验证和危险操作确认。
- 经验内容不包含固定坐标、密码、Cookie、正文或完整截图。
- 实时操控和诊断界面能说明经验是否命中、为何失效以及采用了什么降级路线。
