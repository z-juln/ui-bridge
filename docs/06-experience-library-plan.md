# UI Bridge：全自动操作经验库计划

状态：产品方向已确认，尚未进入实现。经验库同时服务原生 App、网页和内嵌 WebView，
由 UI Bridge 自动管理，不要求用户教学，也不要求 Agent 保存或维护经验。

## 1. 目标

UI Bridge 从已经完成结果验证的真实任务中，自动提取“如何在某类界面中稳定找到目标、
执行动作并确认结果”。后续任务优先复用可信经验，失效时自动降级到通用发现。

经验只能缩短定位过程，不能绕过当前快照、权限、危险操作确认或操作后验证。

## 2. 自动生命周期

1. Agent 正常表达任务意图并使用 Bridge 完成操作。
2. 只有结果状态为 `confirmed` 的操作才可生成候选经验。
3. Bridge 自动去除正文、秘密、临时编号和坐标，提取语义定位、前置条件和验证方式。
4. 候选经验先进入观察状态，在后续界面中做无副作用的影子匹配。
5. 多次独立匹配成功后自动提升可信度并进入优先使用状态。
6. 使用时仍从当前快照重新定位；不确定时自动回退到通用发现。
7. 失败会降低可信度；连续失败后自动停用、重建或替换。
8. 长期未使用、目标版本不再匹配或被新经验替代的记录自动淘汰。

以上过程不弹出“是否保存、更新或删除经验”的日常确认，也不需要独立教学 Skill。

## 3. 经验来源

- 自动经验：从本机已经验证成功的普通任务中生成，是主要来源。
- 官方经验：随版本发布并经过代表环境验证，可自动更新，但必须有可信签名和版本边界。
- 恢复记录：保留有限时间的上一版本，用于新经验导致成功率下降时自动回退。

不同来源不能静默扩大应用范围、网站范围、动作影响或系统权限。官方经验不能覆盖用户本机
更可靠的经验，Bridge 应根据近期验证结果自动选择。

## 4. 一条经验包含什么

- 目标范围：原生应用身份、网站来源、页面类型或 WebView 宿主。
- 操作意图：例如搜索、打开详情、填写收件人或进入设置。
- 使用前条件：窗口或页面应处于什么状态。
- 多组语义定位依据及优先级。
- 允许执行的动作，不保存任意脚本。
- 操作后的预期变化和验证方法。
- 风险类别，以及是否需要前台或人工确认。
- 来源、版本、可信度、最近成功时间和连续失败次数。

原生 App 的定位依据可以包括控件角色、名称、系统标识、窗口类型、父子关系、附近标签和
菜单路径。网页可以包括角色、标签、稳定属性、链接特征、页面区域、iframe 和 Shadow DOM
关系。所有位置都必须从当前快照重新计算。

## 5. 明确不保存的内容

- 密码、Cookie、令牌、验证码或密码类控件值。
- 固定屏幕坐标、临时控件句柄、DOM 顺序或页面节点编号。
- 完整截图、完整 DOM、聊天正文、表单正文或下载内容。
- 能绕过系统权限、网站权限、验证码或危险操作确认的步骤。
- Agent 自己创建的 selector 文件、脚本、memory 笔记或应用专属 Skill。

默认不保存视觉模板。未来确需视觉经验时，只能保存自动脱敏后的局部特征，不能保留原始画面。

## 6. 使用和失效机制

1. 根据当前目标和 Agent 表达的意图查找候选经验。
2. 检查应用、窗口、网站或页面特征是否仍匹配。
3. 用当前快照重新定位，计算可信度和候选控件。
4. 可信度不足时退回通用发现，不盲目执行。
5. 按现有安全规则执行，并重新读取验证结果。
6. 根据结果自动更新可信度、版本和状态。

应用升级、网站改版、窗口变化和页面导航都可能使经验失效。旧经验不能让旧快照、旧坐标
或旧控件句柄继续有效。危险操作经验可以帮助定位，但最终动作仍然每次确认。

## 7. Skill 与 MCP 边界

- 只提供一个日常自动触发的 UI Bridge 控制 Skill，不提供教学 Skill。
- Skill 负责要求 Agent 使用 MCP、读取实时状态、遵守安全规则和验证结果。
- MCP 的普通读取与动作接口在内部自动调用经验引擎，Agent 不需要先调用经验工具。
- 不向 Agent 提供创建、修改、提升、删除或批量导入经验的接口。
- 可提供只读诊断，让 Agent 说明本次是否命中经验、为何降级和使用了什么验证依据。
- 即使 Skill 没有正确加载，Bridge 内部的自动经验生命周期仍然有效。

## 8. 设置与调试界面

设置界面用于透明展示，而不是要求用户日常管理：

- 自动经验数量、适用 App 或网站、可信度和最近成功时间。
- 最近自动创建、提升、降级、替换和淘汰的记录。
- 当前操作是否命中经验，以及为什么降级到通用发现。
- 经验引擎整体开关、清空全部本地经验和恢复上一版本等故障处理入口。

单条经验不要求用户命名、编辑或逐项确认。清空、整体关闭和恢复属于用户主动的故障处理，
仍需二次确认，不能由 Agent 触发。

## 9. 自动质量控制

- 只有完成新状态验证的操作才能训练经验。
- 相同任务至少在独立快照中多次匹配成功后才能成为优先经验。
- 经验使用失败后必须先通用发现，不能对同一目标反复执行。
- 自动评估命中率、误匹配率、验证成功率和降级率。
- 新版本表现变差时自动回滚到上一可靠版本。
- 官方经验更新必须先做影子匹配，不能直接替换本机可靠经验。

## 10. 实施阶段

### 阶段 A：格式、脱敏和自动存储

- 建立跨 App/Web 的统一经验格式、版本、本地加密存储和自动清理。
- 建立敏感信息扫描，确保失败时宁可不学习。

### 阶段 B：原生 App 只读经验

- 从控件查找、读取和导航任务自动生成候选经验。
- 完成影子匹配、可信度、自动升降级和失效降级。

### 阶段 C：原生 App 安全操作经验

- 自动保存动作前条件和结果验证方式。
- 接入前台升级、危险操作确认和连续失败停用。

### 阶段 D：网页与跨通道经验

- Web Bridge 使用同一经验格式和自动生命周期。
- 同一意图可以拥有内嵌浏览器、现有 Chrome 和 WebView 的不同定位方式。

### 阶段 E：官方经验和自动更新

- 建立可签名、可审查、可回滚的官方经验包。
- 更新先影子验证，表现下降时自动恢复。

## 11. 第一轮完成标准

- 用户不教学、不命名、不确认保存，Bridge 能从原生 App 的成功任务自动形成经验。
- 重开窗口后，经验能从新快照缩短定位过程并完成同样的结果验证。
- App 更新或布局变化后，错误经验不会盲目执行并能自动降级、重建或淘汰。
- 经验内容不包含固定坐标、密码、Cookie、正文、完整截图或 Agent 私有脚本。
- Skill 不包含经验管理步骤，MCP 不暴露经验写接口。
- 实时操控和诊断界面能说明经验是否命中、为何失效以及采用了什么降级路线。
