# UI Bridge 当前状态

更新日期：2026-07-15

## 已完成

- 可安装、登录后自动启动的 macOS App。
- 辅助功能和屏幕录制权限检查、缺失提示与系统设置跳转。
- 通用应用、窗口、控件树、窗口截图和控件查找。
- 点击、选择、文本写入、按键、滚动和窗口内坐标操作。
- 每次写操作后的新快照回读与结果确认。
- 过期快照保护、敏感信息遮盖、高影响动作确认和紧急停止。
- 只监听本机的 HTTP 服务和带凭据保护的访问。
- MCP 直接连接与本地服务连接，以及配套通用 Skill。
- Cursor 和 WorkBuddy 的真实写入闭环验收。
- 程序坞图标、菜单栏状态、被操控应用提示和模拟指针反馈。
- 应用、程序、系统身份、连接和本地状态统一使用 UI Bridge 新名称。
- TextEdit、Finder、企业微信和 Electron 类应用的代表性读取或操作验证。
- 七栏原生设置与调试界面，包括多应用实时画面、客户端映射和脱敏诊断报告导出。
- 产品、App、命令、连接、系统身份、启动项和本地状态统一更名为 UI Bridge / `ui-bridge`。
- GitHub 仓库、本地工作区目录和 Codex 项目统一使用 `ui-bridge`。

## 尚未完成

- Windows 版本。
- 浏览器页面和内嵌 WebView 的专用操控通道；方向已确认，技术细节待定。
- 正式开发者签名、公证、安装包、自动更新和公开发布渠道。
- 云端视觉识别和仅靠画面理解复杂自绘界面的能力。
- 团队账号、远程控制、多租户和集中管理。
- 客户端配对和跨客户端的窗口级并发协调。
- 完整的结构化审计日志包；当前已支持不含截图、正文、应用名称和凭据的脱敏诊断报告。
- 为所有第三方应用承诺统一兼容；新应用仍需按代表场景回归。

## 当前限制

- 目前只支持 macOS 14.4 及以上。
- 系统权限必须由用户在 macOS 设置中授权，应用不能代替用户开启。
- 发送、删除、购买、权限变更等高影响动作仍要求人工确认。
- 历史任务编号和 Git 历史保留早期名称，用于审计；当前产品和安装环境不再使用旧名称。
